Unit Testing

A unit test is an automated piece of code that invokes a unit of work in the system and then checks a single assumption about the behavior of that unit of work. A unit of work is a single logical functional use case in the system that can be invoked by some public interface (in most cases).

Example: testing that a transformer function works as expected.

Let's pretend we've built a transformer function that displays the date a dinosaur was created as MM/DD/YY

End to End testing

End-to-end testing involves ensuring that that integrated components of an application function as expected.

Example: testing that a log in process works as expected.

Terms and Definitions

Stubs

A simulated object that mimics the behavior of real objects in a controlled way. It's fake and simply exists to help.

Real Life Example:

I want my UI view to show a dinosuars location.

I'll create a mock object of my dinosaur with coordinates of a location to test my code. This dinosaur simply exists as an object.

Mocks

A simulated object that mimics the behavior of real objects in a controlled way. They're pre-programmed with expectations.

Real Life Example:

I want to test how my ui displays when one of my service calls fails. I'll create a mock of my service, and tell it to return an error to test how my UI handles it.

Spies

An observer. LOL, LIKE A SPY.

Real Life Example:

I want to test that my delete function is called only one when I press the DELETE button rapidly in panic.
